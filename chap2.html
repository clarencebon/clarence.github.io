WEEK 2: System Integration Approach

System Integration

Definition: Is the process of connecting hardware and software elements into a single functional infrastructure, allowing all components to function together.


Integrated System

Definition: assisting organizations in maximizing the benefits from their applications, interconnected apps can quickly find relevant information and work well with essential system components.


Common Types of Integration Systems



Enterprise Application Integration and Data Integration

Definition: It introduces a middleware framework that facilitates seamless data exchange among applications, minimizing the need for extensive database or application modifications. This results in a smoother workflow and enhanced data accessibility, addressing the communication gaps in enterprise software.


Data Integration

Definition: Is the process of merging information from several sources into a single, cohesive picture. It starts with the intake process and includes phases like cleansing, ETL mapping, and transformation. Analytic solutions can only generate useful, actionable business knowledge with the help of data integration.


System Integration Approach



Information-Oriented Approach

Definition: concentrates on integrating several information systems within an organization.

The main objective is to provide smooth data flow and consistent data across diverse platforms.

To facilitate data exchange and accessibility, it involves integrating databases, programs, and data sources.


Information-Oriented Approach



Data Integration

Definition: Integrating databases and data sources to create a unified view of information.


Data Exchange

Definition: Facilitating data exchange between systems using standard data formats or APIs.


Data Consistency

Definition: Ensuring that data remains consistent and up-to-date across integrated systems.


Reduced Data Redundancy

Definition: Eliminates data duplication, leading to a single source of truth.


Improved Decision Making

Definition: Access to real-time, accurate data for better decision-making processes.


Complexity

Definition: Integrating diverse data sources with varying formats can be complex.


Data Integrity

Definition: Data inconsistencies or errors in one system can affect others if not properly managed.


Business Process Integration-Oriented Approach


Aims to integrate business operations across many systems.

To accomplish end-to-end process automation, it includes connecting a number of apps and workflows.

The objective is to increase productivity, decrease manual involvement, and improve business processes.



Workflow Integration

Definition: Integrating applications and systems to support seamless process flows.


Business Rules

Definition: Implementing business rules across systems to maintain consistency.


Process Automation

Definition: Automating tasks and approvals to reduce manual effort and processing time.


Enhanced Efficiency

Definition: Automated processes lead to faster and error-free task execution.


Streamlined Workflows

Definition: Integration ensures a smooth flow of information between different steps.


Improved Collaboration

Definition: Facilitates collaboration among departments and teams.


Dependency on Applications

Definition: Changes in one application may affect the entire integrated process.


Point-to-Point Integration Approach


Represents a straightforward and direct method.

Encompasses the direct interconnection of individual systems.

Suitable for small-scale integration's involving a few systems.

May evolve into a complicated and demanding task to uphold as the quantity of integration's rises.



Service-Oriented Integration Approach


is based on the principles of Service-Oriented Architecture (SOA).

Services are self-contained and modular elements with standardized interfaces.

Communication between systems occurs through the invocation of these services across a network.

This framework facilitates integrations with traits like re-usability, flexibility, and scalability.



Benefits of Service-Oriented Integration Approach



Loose Coupling

Definition: Services are independent, allowing changes without affecting other systems.


Flexibility

Definition: New services can be added or modified to accommodate changing business needs.


Reusability

Definition: Services can be used in multiple integrations, reducing redundancy.


Portal-Oriented Integration Approach


Revolves around employing a central portal as an intermediary tier between systems.

The portal controls how integrated systems communicate with one another and exchange information.

Applicable in situations where numerous systems necessitate access to a common pool of data or functionalities.



Benefits of System Integration Approach



Improved Data Accuracy

Definition: Information flows seamlessly, reducing data inconsistencies.


Enhanced Efficiency

Definition: Streamlined processes lead to increased productivity.


Better Decision Making

Definition: Access to real-time data for informed decision-making.


Flexibility and Scalability

Definition: Systems can adapt to changing business needs.


Enhanced User Experience

Definition: A unified interface provides a smooth user experience.
